  我们的理解
我们能够理解开发者在收到垃圾邮件的愤怒和无奈，这次的确是不仅在手段上亦或是在后续的回应上都有不对的地方。我们后续会杜绝此类事件事件的再次发生，将更符合开发者社区更为认同的方式。

非情绪化质疑及回应
    关于想赚钱的心情，和道德准绳（亦或是准则）高低的质疑
    答：很难将两者画一个等号，去进行此高彼低，此低彼高的评价。不见得赚钱了准绳就高，不赚钱准则就低。现在社区还一直处于一个非盈利的阶段，我们也不着急于商业化，因为还有一些疑问需要解决。关于后者来说，人非圣贤也又脑袋一热的时候。在此感谢各位开发者的愤怒和指责，也希望能够更加的非情绪化的指摘
    关于公开邮箱的使用
    答：关于公开邮箱的使用，我也是通过这样的机会加入到春松客服这个组织下的。我个人选择是多一个机会，虽然垃圾邮件让我也很苦恼，但是我也仍然开放的将邮箱公开，能够看到一些不一样吧。可能我这么说，是不是我的邮箱也会被爆破了？让我也感受一下垃圾邮箱的威力~
作为此次事件的被告方，我们更应该理解开发者的诉求和意愿。这点我们做错了，我们会找到更好的方式来与开发者之间建立联系。也不希望通过这样的方式认识你们，将来的我们。
    开源承诺的质疑
    答：现在关于春松客服系统的协议，我们是基于 Apache 2.0 的基础之上演化的。也是希望对于未来商业化的可能性至少得给参与者一个交代。我们会承诺代码肯定是开源可审计的前提下，逐步思考开源商业化的可能性。现在这个开源协议既不实际可能也可能有不得体的地方。对于公众的疑问，我们欢迎质疑并尽力告知我们想做的是一件什么事情，而不仅仅只是被动理解下的一种“诡辩般”的解释

<details><summary>关于春松客服 SPAM 事件的回顾</summary>
一、起因
春松客服公开抓取 GitHub 用户邮箱并发送垃圾邮件, 官方仓库反馈后遭到 PUA "受害者有罪论", 且项目管理者 hailiang-wang 认为 "在 GitHub 公开邮件即代表你的邮箱愿意接受垃圾邮件, 同时不接受视为打消他们的热情".
此后我在 V2EX 发帖表示怀疑, 见 t/969939.
https://web.archive.org/web/20230901022907/https://github.com/cskefu/cskefu/issues/931
二、经过
V 站网友基本一致性表示做法有问题, 并建议举报该仓库和其作者; 经过本人思考后觉得没有什么问题, 毕竟该作者已经属于 SPAM 应该承担责.
三、重点
这里说一下整件事情让 [春松客服] "遭人讨厌" 的核心问题, 也是我本人认为最核心的的问题:
    1 、其项目作者在看到 V 站帖子后丝毫不觉得有问题, 且还进行引流, 见 t/970167
    2 、经网友发现其官网公开 网暴 一个很普通的 GitHub 用户, 且对其微信骚扰直至自行删号; 全程对此用户各种辱骂侮辱性词汇, 见 Wayback Machine
    3 、在观看春松客服官网文章, 了解 被网暴用户 的前因后果后发现这个用户完全没有做错任何事
    4 、整场事件中出现了两个所谓的春松客服社区人员, 两个人虽然有道歉声明但反复切换身份, 我个人甚至不清楚你们到底是想把社区和 hailiang-wang 分开还是想博同情? 甚至道歉在我看来没有丝毫诚恳!
    5 、整场事件持续 1 天的时间, 从未见过 hailiang-wang @hain 发出过一句道歉的话! 事件发酵时间这么久被网暴用户的帖子昨晚才从官网删除!

这本就是一件 "抱歉打扰到您, 我们今后不会再发送垃圾邮件" 的事, 但是整件事件透露着 hailiang-wang 这个人到现在都认为毫无问题、用户可以随意被践踏、别人可以随意网暴的操作. 我不清楚这个人在 [春松客服] 社区到底扮演着什么样的角色, 但是我个人认为:
1 、如果你们作为社区运作, 那么这种显而易见的过失行为应该最短时间内处理, 并做出对事件负责人的处罚以及说明, 而不是来 V 站 battle

3 、如果你们作为社区运作, 一个人搞出这么恶劣影响的事情你们至少也应该有能力让他发声道歉或者直接公布社区处罚结果
